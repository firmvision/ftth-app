1. Admin Access & Security

1.1 Logging In

Navigate to the platform URL in any modern browser. Click the "Admin / Teacher" tab on the login page. Use your administrator credentials.

Security: The admin account has unrestricted access to all data. Never share your credentials. Log out (top-right menu → Logout) whenever leaving the computer unattended.

1.2 First Login — Change Default Password

The system is seeded with a default admin account. Change the password immediately after first login:

Go to Students page → find "admin" user → Edit → change password to a strong unique password (12+ characters, mix of letters, numbers, symbols).

1.3 Admin Credentials Management

Store admin credentials in a secure, offline location (locked cabinet). Do not store in email, WhatsApp, or unencrypted digital files. Rotate passwords at the start of each academic year.

StudentView available exams, take exams, view their own resultsAccess admin panel, see other students' answers

2.2 Adding a Single User

Go to Students → click "+ Add Student" (or "+ Add Teacher").
Fill in: Full Name, Username, Password, Class/Arm, Role.
Username convention: Use firstname.lastname format (e.g., adewale.johnson) for easy identification. Avoid spaces or special characters in usernames.

2.3 Bulk Student Import (CSV)

For importing large numbers of students at once:

  1. Download the CSV Template from the Students page (or use business/student-import-template.csv).
  2. Fill in columns: full_name, username, password, class, arm
  3. Save as .csv (UTF-8 encoding).
  4. Go to Students → Bulk Import → upload the file.
  5. Review the preview table → click "Confirm Import".
Password Note: Students' initial passwords are set from the CSV. For security, require students to change passwords on first login, or issue new passwords each exam period via a new CSV bulk reset.

2.4 Editing & Deleting Users

  • Edit: Click the pencil icon next to any user. Change any field and save.
  • Delete: Click the trash icon. Deletion removes all associated exam records — this is irreversible. Consider deactivating (disabling) students instead of deleting to preserve historical data.
  • Password Reset: Edit the user and enter a new password.

2.5 Teacher Accounts

Create teacher accounts with Role = "Teacher". Teachers see the same admin panel but with restricted access. Assign teachers to their subjects by ensuring they only create questions for their assigned subjects.

3. Subjects & Classes

3.1 Creating Subjects

Go to Subjects → click "+ Add Subject" → enter the subject name (e.g., "Mathematics", "English Language") → Save.

Create all subjects before adding questions. Subjects are shared across all teachers and classes.

3.2 Class Organisation

Classes are set at the student level (e.g., SS1, SS2, JSS3) and the exam level. An exam set for "SS2" will only be visible to students whose account has class = "SS2". Use consistent naming across all records.

4. Question Bank Administration

4.1 Question Review Process

Establish a review workflow before exam season:

  1. Teachers submit questions (they can add to the bank directly)
  2. Admin reviews for accuracy, clarity, and appropriate difficulty
  3. Approved questions are included in exam assemblies

4.2 Question Organisation

Filter questions by subject, difficulty, or class when building exams. Use the Difficulty field (Easy / Medium / Hard) to maintain a balanced distribution across exam papers.

4.3 Editing & Deleting Questions

Warning: Do not edit or delete questions that are part of an active or completed exam. Doing so can corrupt scoring for sessions that used that question. Archive completed exams before modifying question bank entries.

5. Exam Lifecycle

5.1 Status Flow

StatusWhat it MeansWho Can See the Exam
DraftBeing prepared; no students can see itAdmin and assigned teacher only
ScheduledWill auto-activate at the set start timeOnly if past start time
ActiveOpen for students to takeStudents whose class matches
EndedClosed; no new attempts; results viewableAdmin and teacher (results only)

5.2 Pre-Exam Checklist

  • ☐ All questions reviewed and verified
  • ☐ Correct class assigned to exam
  • ☐ Duration set correctly (in minutes)
  • ☐ Pass mark confirmed with academic team
  • ☐ Max violations configured (recommend: 3)
  • ☐ Exam tested by admin with a test student account
  • ☐ All students' accounts verified (class/arm correct)
  • ☐ Invigilators briefed on procedures
  • ☐ Server health confirmed (check /api/health endpoint)

5.3 During the Exam

Monitor active sessions from the Dashboard. The "Active Sessions" count shows how many students are currently in the exam. If a student has a technical problem:

  1. Ask the student to log back in — their session resumes automatically
  2. If session is corrupted, you can reset it via: Results → Student Session → Reset (contact Firmvision support if option is not visible)

5.4 Ending an Exam

Click "End Exam" on the Exams page. All in-progress sessions are NOT auto-submitted — students currently in the exam can complete it until their timer expires. After the timer expires, sessions auto-submit.
Best practice: Announce "Exam ending in 5 minutes" before clicking End Exam, so invigilators can ensure all students save their last answers.

6. Results & Analytics

6.1 Accessing Results

Go to Exams → View Results (📊 icon) next to any completed exam.

6.2 Understanding the Analytics Dashboard

MetricDescription & Use
SubmissionsTotal exam sessions submitted. Compare with enrolled students to identify absentees.
Pass Rate% of students who met the pass mark. Below 50% pass rate may indicate question difficulty or preparation issues.
Class AverageMean score %. Compare across classes or exam periods to track improvement.
Score DistributionBar chart of score bands. A bell curve indicates good question calibration. Skewed right = too easy; left = too hard.

6.3 Session Detail View

Click any student's name to see their full answer breakdown. Useful for:

  • Investigating a student's score dispute
  • Checking violation log to verify anti-cheat flags
  • Identifying questions many students answered wrongly (possible ambiguity)

7. Backup & Recovery

7.1 Running a Manual Backup

On the server, run: bash /path/to/cbt/backup.sh backup

Backups are saved to cbt/backups/ as compressed archives. Each includes the database and application files.

7.2 Automated Daily Backups

Set up automatic daily backup at 2 AM: bash /path/to/cbt/backup.sh cron

7.3 Restoring from Backup

bash /path/to/cbt/backup.sh restore — restores from the most recent backup.
Warning: Restore overwrites current data. Only perform after confirming with Firmvision support. Always take a fresh backup before restoring.

7.4 Backup Storage Policy

  • Backups older than 30 days are automatically deleted
  • Copy at least one backup per month to an external storage (USB or cloud drive)
  • Verify backup integrity quarterly by doing a test restore on a staging server

8. Security Practices

  • Change default passwords immediately after installation
  • Rotate all staff passwords at the start of each academic year
  • Limit admin accounts — only the IT Administrator and Deputy Principal should have admin role
  • Review teacher accounts — disable teachers who have left the school
  • Keep the server updated — apply OS and platform security patches when notified by Firmvision
  • Restrict physical server access — server room should be locked; only IT staff should have access
  • Monitor the violation log during exams — unusual patterns may indicate organised malpractice
  • Exam content is confidential — do not share question bank access outside designated staff

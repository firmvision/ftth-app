1.3 Network Requirements

  • Server must have a static IP address on the LAN (e.g., 192.168.1.100)
  • All student devices must be on the same network or VLAN as the server
  • No internet required for LAN-only deployment during exams
  • Open port: 80 (HTTP) or 443 (HTTPS) on the school network firewall

3. Server OS Setup

3.1 Fresh Ubuntu Server Installation

Download Ubuntu Server 22.04 LTS from ubuntu.com and install on your server hardware. During setup:
  • Set server name: cbt-server
  • Create admin user: cbtadmin
  • Enable OpenSSH server
  • Do NOT install additional software (base install only)

3.2 Set Static IP Address

# Edit Netplan configuration
sudo nano /etc/netplan/00-installer-config.yaml

# Set content like this (adjust interface name with: ip link show):
network:
  version: 2
  ethernets:
    enp0s3:               # Replace with your interface name
      dhcp4: no
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

sudo netplan apply
ip addr show  # Verify static IP

4. Automated Installation (Recommended)

The setup.sh script automates the entire installation:

# 1. Copy the CBT project to the server (from USB or scp)
scp -r cbt/ cbtadmin@192.168.1.100:/home/cbtadmin/

# 2. SSH into the server
ssh cbtadmin@192.168.1.100

# 3. Run the setup script as root
cd /home/cbtadmin/cbt
sudo bash setup.sh

# For cloud deployment with a domain name:
sudo bash setup.sh cbt.yourschool.edu.ng
What setup.sh does automatically:
  • Installs Python, PostgreSQL, Nginx, and all system dependencies
  • Creates the database, user, and generates a secure random password
  • Creates a Python virtual environment and installs Python packages
  • Generates a .env file with secure secret keys
  • Creates and enables a systemd service for automatic startup
  • Configures Nginx as a reverse proxy
  • Sets up UFW firewall rules
  • Displays admin credentials at the end
Save the credentials shown at the end of setup.sh immediately. They are not stored anywhere else. The admin password is a random generated string.

5. Manual Installation (If Needed)

# Install system packages
sudo apt-get update && sudo apt-get install -y \
    python3 python3-pip python3-venv python3-dev \
    nginx postgresql postgresql-contrib libpq-dev

# Set up PostgreSQL
sudo -u postgres psql -c "CREATE USER cbt WITH PASSWORD 'your_secure_password';"
sudo -u postgres psql -c "CREATE DATABASE cbt_exam OWNER cbt;"

# Create Python venv and install dependencies
cd /path/to/cbt && python3 -m venv venv
source venv/bin/activate
pip install -r backend/requirements.txt

# Create .env file
cat > backend/.env << 'EOF'
DATABASE_URL=postgresql://cbt:your_secure_password@localhost:5432/cbt_exam
SECRET_KEY=replace_with_long_random_string_at_least_48_chars
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_admin_password
CORS_ORIGINS=*
EOF

# Set up static files
mkdir -p static && cp login.html admin.html student.html static/

# Test run (Ctrl+C to stop)
cd backend && uvicorn main:app --host 0.0.0.0 --port 8000

6. Nginx Configuration

# /etc/nginx/sites-available/cbt-exam
server {
    listen 80;
    server_name 192.168.1.100;  # Or your domain name

    client_max_body_size 20M;

    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
    }
}

sudo ln -s /etc/nginx/sites-available/cbt-exam /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

7. HTTPS / SSL Setup (Optional but Recommended for Cloud)

# Install Certbot (requires internet + domain pointing to server)
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d cbt.yourschool.edu.ng

# Auto-renewal is configured automatically by Certbot
sudo certbot renew --dry-run  # Test renewal
LAN-only deployments: HTTPS is not strictly required when traffic stays within the school LAN. HTTP on port 80 is sufficient for offline LAN deployments. Use HTTPS for cloud/internet-facing deployments.

8. Firewall Configuration

# UFW setup (done by setup.sh — manual steps shown here)
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status verbose

9. Backup Configuration

# Run a manual backup now
bash /path/to/cbt/backup.sh backup

# Set up daily automatic backup at 2 AM
bash /path/to/cbt/backup.sh cron

# List available backups
bash /path/to/cbt/backup.sh list

# Restore from a specific backup file
bash /path/to/cbt/backup.sh restore --file backups/cbt_backup_20260520_020000.tar.gz
External backup: Copy backup archives to an external USB drive or network storage weekly. Store at least 3 months of backups off-site (e.g., on a school cloud drive).

10. Monitoring & Logs

# Check service status
systemctl status cbt-exam

# View live application logs
journalctl -u cbt-exam -f

# View last 100 log lines
journalctl -u cbt-exam -n 100

# View Nginx access/error logs
tail -f /var/log/nginx/cbt-exam-access.log
tail -f /var/log/nginx/cbt-exam-error.log

# Check API health endpoint
curl http://localhost:8000/api/health
# Expected: {"status": "ok"}

# Restart the service
systemctl restart cbt-exam

# Check disk space
df -h

11. Docker Deployment (Cloud)

# Prerequisites: Docker + Docker Compose installed
cd /path/to/cbt

# Set environment variables
export SECRET_KEY="your_very_long_random_secret_key_here"
export ADMIN_PASSWORD="YourStrongAdminPassword"

# Start all services (app + PostgreSQL)
docker compose up -d

# Check status
docker compose ps
docker compose logs -f cbt-api

# Stop services
docker compose down

# Restart after update
docker compose down && docker compose up -d --build

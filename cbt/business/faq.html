CBT Exam Platform — Pre-Sales & Technical Guide

General Questions

What is the Firmvision CBT Platform? ▶

Firmvision CBT Platform is a complete Computer-Based Testing solution designed specifically for Nigerian secondary schools. It allows schools to create, administer, and automatically grade examinations online or offline on a school LAN — with robust security features to prevent cheating.

Does the system require internet access during exams? ▶

No. This is one of the platform's core advantages. After installation, the system runs entirely on your school's local area network (LAN). Students access it via the school's Wi-Fi or wired network from any device — no internet connection required during examinations.

Internet is only needed for initial installation, software updates, and (optionally) cloud backup.

How many students can use the platform simultaneously? ▶

The platform is built to support 5,000+ students enrolled, with hundreds taking exams simultaneously — depending on your server specifications. On a recommended server (8 cores, 16 GB RAM), 200–500 concurrent exam sessions run smoothly.

For larger concurrent loads, we provide guidance on server scaling and load balancing.

What devices can students use? ▶

Any device with a modern web browser:

  • Desktop / laptop computers (Windows, macOS, Linux)
  • Android tablets and smartphones
  • iPads and iPhones
  • School Chromebooks

No app installation is required — students open a web browser and navigate to the server address.

Exam Security & Integrity

How does the system prevent cheating? ▶

Four layers of anti-cheat protection are built in:

  1. Fullscreen Lock: The exam browser is forced into fullscreen mode. Exiting triggers a warning and violation count.
  2. Tab-Switch Detection: If a student switches browser tabs or minimises the window, this is detected and logged immediately.
  3. Question Randomisation: Questions and answer options are shuffled differently for every student — making copying nearly impossible.
  4. One-Time Session Tokens: Each exam session uses a unique cryptographic token. Sessions cannot be shared or used by another student.

If a student triggers too many violations, the exam is automatically submitted on their behalf.

What happens if a student's power or network cuts during an exam? ▶

The platform uses continuous auto-save. Every answer is saved to the server the moment the student selects it. If their device loses power or connection:

  • All saved answers are preserved
  • The student can reconnect and resume from where they left off
  • The exam timer continues server-side — it cannot be paused by disconnecting

The administrator can also manually reset a session if there is a genuine infrastructure failure.

Can students share their login credentials to have someone else take the exam? ▶

The one-time session token system significantly mitigates this risk. Each exam session is bound to a single login token that cannot be reused. Schools are also advised to:

  • Seat students in supervised examination halls
  • Issue unique, random passwords per exam period (bulk-resettable via admin panel)
  • Verify student identity before allowing them to log in

Administration & Setup

How do we add 5,000 students to the system? ▶

The Admin panel includes a bulk CSV import feature. You prepare a spreadsheet with columns: full_name, username, password, class, arm and upload it — all students are created instantly.

A filled template CSV with sample Nigerian student names is provided. The import validates data before processing and reports any errors.

What question types does the system support? ▶
  • Multiple Choice (MCQ): One correct answer from 2–6 options
  • True / False: Binary choice, auto-scored
  • Multi-Select: Multiple correct answers; partial credit available
  • Essay / Long Answer: Open text; manually graded by teacher

All question types can be mixed within the same exam.

Can teachers create their own questions and exams? ▶

Yes. Teachers are given a dedicated login with permission to:

  • Add and edit questions in the question bank (their subjects only)
  • Create exams and assign questions
  • View results and grade essay answers

Teachers cannot manage students, access other teachers' questions, or change system settings — those are restricted to admins.

How long does installation take? ▶

A standard installation on a prepared server takes 2–4 hours, including:

  • Operating system and dependency setup
  • Platform installation and database configuration
  • Network access testing from student devices
  • Admin credential setup and initial data import

The automated setup.sh script handles most of this. Full deployment including training typically takes 1–2 days on-site.

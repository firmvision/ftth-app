Section 1

Executive Summary

Firmvision Technologies Limited is pleased to present this proposal for the design, development, and full deployment of a secure, enterprise-grade Computer-Based Testing (CBT) Platform tailored exclusively for Babcock University High School.

As one of Nigeria's leading faith-based academic institutions, Babcock University High School maintains rigorous academic standards across a student population of approximately 1,500 learners — a number expected to grow beyond 2,000 in the coming academic years. Managing internal assessments, continuous assessment tests (CATs), terminal examinations, and promotional examinations at this scale — while preserving integrity, reducing administrative burden, and generating timely results — requires a purpose-built digital solution.

Our proposed Babcock CBT Platform delivers a fully-featured examination environment with an administrative control panel for teachers and management, a secure student examination portal, a centralised question bank, automated marking, and a robust results management system. The platform is engineered to operate on the school's existing local area network (LAN) without any dependency on internet connectivity — and can equally be accessed over the internet when required, giving the institution the flexibility of a hybrid deployment model.

🎯
Our Core Promise A tamper-proof, efficient, and cost-effective CBT system that eliminates paper-based exam challenges, reduces result-processing time from days to minutes, and gives Babcock University High School full ownership and control of its academic assessment data.

Section 4

Proposed Solution Overview

Firmvision Technologies proposes the development and deployment of the Babcock CBT Platform — a purpose-built, offline-capable, internet-ready Computer-Based Testing system designed specifically for Babcock University High School.

The platform consists of three integrated components working seamlessly together:

🛡️
Student Examination Portal
A secure, locked-down examination interface for students. Features fullscreen enforcement, tab-switch detection, countdown timer, auto-save, and automated submission. Designed to work on any modern browser — no app installation required on student devices.
👩‍💼
Admin & Teacher Control Panel
A comprehensive management dashboard for school administrators and teachers. Manage students in bulk, build and organise the question bank, create and schedule examinations, monitor live sessions, view detailed results, and manually grade essay questions.
⚙️
Secure Backend API
A high-performance Python FastAPI backend with role-based authentication, encrypted data storage, question randomisation engine, automatic grading, violation logging, and support for both SQLite (offline/LAN) and PostgreSQL (cloud/enterprise) databases.
📦
Zero-Dependency Design
Unlike platforms requiring constant internet access, all platform components — including the admin panel and student portal — function entirely from the school's local server. No third-party CDN, cloud service, or internet connection is required during examination sessions.
Section 5

Key Platform Features

Question Bank Management

A centralised, subject-organised repository of examination questions that teachers build once and reuse across multiple examinations.

  • Four question types: Multiple Choice (MCQ), True / False, Multi-select, and Essay / Short Answer
  • Questions tagged by subject, class level, and topic
  • Configurable point values per question
  • Optional explanations for each question — displayed after grading
  • Full-text search and filter by subject, type, or keyword

Examination Management

  • Create exams from question bank with question selection and ordering controls
  • Set time limits, start/end windows, and target class groups
  • Configurable pass mark percentage per exam
  • Questions and answer options shuffled uniquely per student
  • Pre-exam instruction screen — students are shown a six-step briefing (identity verified, navigation guide, timer rules, fullscreen policy, submission procedure, result display) before the exam begins, ensuring no student can claim ignorance of the rules
  • Exam status workflow: Draft → Active → Ended

Student Management

  • Register students individually or bulk-import up to 2,000+ via CSV upload
  • Students organised by class, registration number, and academic level
  • Role-based access: Admin, Teacher, Student
  • Account enable/disable without data deletion
  • Secure password management with bcrypt encryption
  • Multi-device support — the student examination portal is fully optimised for desktop computers, laptops, tablets, and smartphones. Mobile users receive a dedicated bottom navigation bar (Previous / Question Counter / Next / Submit) with touch-friendly answer buttons, ensuring examinations can be conducted even in environments where desktop computers are limited

Results & Analytics

  • Instant automatic grading for MCQ, True/False, and Multi-select questions
  • Manual essay grading interface with partial marks support
  • Per-student result cards showing score, percentage, time taken, and violation log
  • Class-wide results table sortable by score, name, or class
  • Average score, pass rate, and highest/lowest scores per examination
  • Score distribution analytics chart — a colour-coded bar chart (red below 50%, amber 50–69%, green 70%+) showing how the class performed across ten percentage bands, giving teachers an instant visual overview of examination difficulty and class preparedness
  • Printable student result slip — immediately after submission, students can print a formal result slip showing their name, examination title, score, percentage, pass/fail status, date and time, and signature lines for school stamp and countersignature — eliminating the need to manually transcribe results onto paper
  • Export-ready data for school record systems
Section 6

Security & Examination Integrity

Examination integrity is the cornerstone of any credible assessment system. The Babcock CBT Platform employs a multi-layered security architecture at both the technical and procedural level.

🖥️
Fullscreen Enforcement The exam automatically requests fullscreen mode on start. Exiting fullscreen logs a violation and triggers a warning. Repeated exits auto-submit the exam.
🔁
Tab-Switch Detection Any navigation away from the exam tab or window is instantly detected, logged, and reported. Students are warned; repeated switches trigger auto-submission.
🎲
Question & Option Randomisation Each student receives questions and answer options in a unique random order, making sharing answers between adjacent students impossible.
🎫
One-Time Session Tokens A unique cryptographic token is issued per student per exam. The same student cannot open the same exam on two different devices simultaneously.
🚫
Copy & Right-Click Disabled During active examinations, clipboard access, right-click menus, and browser developer tools are blocked to prevent question extraction.
⏱️
Server-Authoritative Timer Exam time is tracked server-side and verified via regular heartbeat calls. Manipulating the browser clock has no effect on the exam timer.
🔐
JWT Authentication & Bcrypt All sessions use industry-standard JSON Web Tokens. Passwords are stored using bcrypt hashing — never in plain text. All API endpoints are protected.
📋
Violation Audit Log Every security event is timestamped and stored against the student's session. Administrators can review the full violation history for any submission.
Section 7

Deployment Options — Offline & Online

A key differentiator of the Babcock CBT Platform is its ability to operate in both offline (LAN) and online (internet) environments without any code changes. The school can choose either mode or run them simultaneously.

Mode 1 — Recommended
🏫 School LAN (Offline)

The platform server runs on a single school computer or dedicated server. All students connect through the school's Wi-Fi or LAN cable. No internet access required at any point.

  • Works on existing school Wi-Fi infrastructure
  • All data stays within the school premises
  • SQLite database — zero additional software
  • Supports 500+ concurrent users on a standard PC
  • Single command to start: ./start.sh
  • Ideal for terminal exams and internal assessments
Mode 2
🌐 Cloud / Internet

Deployed on a cloud server (e.g. a VPS or dedicated server), the platform is accessible from anywhere — school, home, or mobile — over the internet.

  • Accessible from any device, anywhere
  • PostgreSQL database for high-volume concurrency
  • Docker deployment for easy management
  • Encrypted HTTPS for all data in transit
  • Suitable for remote/take-home assessments
  • Daily automated database backups
💡
Hybrid Recommendation For maximum reliability, we recommend the LAN-primary, internet-backup model: conduct all formal examinations on the school's local network (zero dependency on internet provider), while keeping the cloud deployment available for remote access by school management and reporting.

Technical Architecture

The platform is built on a modern, proven technology stack selected for performance, security, long-term maintainability, and independence from expensive proprietary software licenses.

Section 9

Implementation Timeline

From contract signing to full go-live, the project is structured into four phases over an estimated 6–8 weeks.

Weeks 1 – 2
Phase 1 — Discovery & Setup
Requirements Finalisation & Infrastructure Setup
Kick-off meeting with school management and IT team. Finalise subject list, class structure, user roles, and examination policies. Assess existing network infrastructure. Set up the server environment (school-provided hardware or cloud, per agreement). Install and configure all platform components. Create initial admin accounts.
Weeks 3 – 4
Phase 2 — Data Migration & Configuration
Student Registration & Question Bank Loading
Bulk import of all student records (name, class, registration number) from school register. Subject configuration for all classes and academic levels. Teachers receive accounts and begin loading questions into the bank with Firmvision technical support on-site. Platform customised with school branding (name, logo, colour scheme).
Weeks 5 – 6
Phase 3 — Training & Pilot Testing
Staff Training & Mock Examination
Structured training for administrators, teachers, and IT staff on all platform functions. Printed user guide and quick-reference cards for teachers. Conduct a supervised pilot examination with a sample student group (one class or year group) to validate the full workflow from question loading to result generation. Feedback incorporated and issues resolved.
Weeks 7 – 8
Phase 4 — Go-Live & Handover
Full Deployment & Official Launch
Full school-wide examination conducted on the platform. Firmvision technical team present on-site for the first official examination cycle. Formal handover documentation, admin credentials, backup procedures, and disaster recovery guide delivered. 90-day intensive post-launch support period commences.
